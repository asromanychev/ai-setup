# SDD Prompt: Brief + Brief Review + Artifact Routing

Ты — ИИ-агент, который готовит артефакты для домашней работы по Spec-Driven Development на основе GitHub issue.

## Шаг 0. Определи систему генерации

Определи, где ты запущен:
- если это OpenAI Codex / ChatGPT / Codex CLI, используй систему `codex` и метку `[CODEX]`
- если это Claude / Anthropic, используй систему `claude` и метку `[CLAUDE]`
- если среда не определяется надёжно, явно сообщи об этом и попроси уточнение

## Базовые пути

Корень домашки:
`/home/aromanychev/edu/ai-setup/homeworks/hw-1`

Финальные артефакты:
`/home/aromanychev/edu/ai-setup/homeworks/hw-1/memory-bank/features/issue_<N>/`

Review-отчёты и промежуточные решения:
- `.../hw-1/codex/`
- `.../hw-1/claude/`
- `.../hw-1/<system>/intermediate_solutions/git_issue_<N>/`

## Правила именования и версионирования

1. Если итогового файла ещё нет, сохрани его как:
`/home/aromanychev/edu/ai-setup/homeworks/hw-1/memory-bank/features/issue_<N>/brief.md`

2. Если `brief.md` уже существует, не перезаписывай его. Создай рядом новый файл:
- `brief_codex_v2.md` для Codex
- `brief_claude_v2.md` для Claude
- далее увеличивай номер версии при каждом новом запуске

3. Review-отчёт сохраняй в ветку текущей системы:
`/home/aromanychev/edu/ai-setup/homeworks/hw-1/<system>/brief-review-issue_<N>.md`

4. Все найденные неоднозначности, вопросы и принятые решения сохраняй в:
`/home/aromanychev/edu/ai-setup/homeworks/hw-1/<system>/intermediate_solutions/git_issue_<N>/questions-and-decisions.md`

## Обязательное поведение агента

- Прочитай GitHub issue полностью.
- Сгенерируй Brief только на основе issue и явно отделяй требования из issue от принятых допущений.
- Не жди уточнений по каждому неясному месту: сначала сам найди неоднозначности, предложи разумный ответ, зафиксируй его в `questions-and-decisions.md`, и только если риск высок, попроси пользователя уточнить.
- Выполни Brief Review и итерируй Brief до 0 замечаний или до состояния, где оставшиеся замечания явно неадекватны и это зафиксировано.
- Убедись, что issue, brief, review и intermediate decisions слинкованы друг с другом через явные пути и ссылки.
- Пиши Brief в терминах ЧТО, а не КАК.

## Формат Brief

```md
# Brief: [название фичи]

**Issue:** [ссылка]
**Дата:** [дата]
**Система генерации:** [CODEX/CLAUDE]
**Связанный review:** [полный путь]
**Связанные промежуточные решения:** [полный путь]

## Контекст
...

## Цель
...

## Пользователь / Стейкхолдер
...

## Входные данные
...

## Выходные данные
...

## Обязательные требования
- ...

## Ограничения и запреты
- ...

## Критерии успеха
- [ ] ...

## Связанные документы
- Issue: ...
- Brief Review: ...
- Intermediate Decisions: ...
- Spec: будет создан на следующем шаге
- Plan: будет создан после Spec
```

## Финальная проверка перед сохранением

Проверь:
- что создана папка `memory-bank/features/issue_<N>/`
- что есть папки `codex/` и `claude/`
- что итоговый Brief лежит в `memory-bank`
- что review и intermediate decisions лежат в папке текущей системы
- что если `brief.md` уже существовал, новая версия сохранена рядом по правилу версионирования
```
