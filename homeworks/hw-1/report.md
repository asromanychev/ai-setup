## Дырки в документах

### В Brief не хватало
- Явного перечисления всех обязательных полей результата одного шага синка.
- Явной фиксации edge case для неизвестного `plugin_id` в измеримых критериях.

### В Spec была неоднозначность
- Нужно ли считать checkpoint opaque payload или структурированным `Hash`.
- Нужно ли допускать дополнительные поля в результате шага поверх минимального набора.

### В Plan не был учтён
- Явный пункт на проверку актуальности версий GitHub Actions.
- Явный пункт на проверку согласованности путей артефактов домашки (`001` против `issue_1`).

## Остаточные риски по сдаче

- По формулировке домашки ожидается 3–10 повторов цикла, а в текущей сдаче оформлена только одна фича. Для проверки именно `issue_1` пакет теперь в порядке, но для полного соответствия недельному объёму потребуется ещё 2+ итерации.
