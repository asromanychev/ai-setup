# Стиль кода и границы ответственности

Краткий SSoT по договорённостям разработки; детали и снятия cops — в Aitlas и Rubocop-конфиге репозитория.

## Обязательное

- **Rubocop** — новый код проходит проверку; не ослаблять cops без обоснования в конкретном месте.
- **Явные service objects** и небольшие классы с одной зоной ответственности.
- **Ядро и плагин не смешивать:** плагин знает источник; ядро — оркестрацию, хранение, прогресс.
- **Адаптеры / клиенты** внешних API: только получение, маппинг в сырой формат, передача на сохранение — без интерпретации смысла контента.

## Где полные правила

- Профилированные правила Cursor: `aitlas/ai-da-collect/cursor/rules/` (symlink из `.cursor/rules/` при наличии)
- ADR и соглашения: `aitlas/ai-da-collect/conventions/docs/`

## Thread safety (Ruby)

- Не использовать `cattr_accessor`, `@@`, глобальные `$` для изменяемого состояния запроса.
- Мутабельные объекты в константах не хранить.
