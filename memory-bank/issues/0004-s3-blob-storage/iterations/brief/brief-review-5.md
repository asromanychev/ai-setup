# Ревью Brief v4 — Issue #4: S3-совместимый адаптер для raw blob

## Замечание 1 (Неблокирующее) — Размытая граница в `Что не в скоупе`

**Цитата:**
> Изменение доменной модели PostgreSQL сверх необходимости ссылаться на уже сохранённый blob.

**Проблема:**
Формулировка `сверх необходимости` не задаёт однозначной границы. Из текста нельзя проверить, какой именно объём изменения доменной модели считается допустимым, а какой уже выходит за scope. Для раздела `Что не в скоупе` это плохая формулировка: он должен закрывать пространство для домысливания, а не оставлять место для разных трактовок у следующего артефакта.

Это не блокирует понимание основной проблемы и не ломает success criteria, но оставляет нестрогую границу задачи именно в том разделе, который должен быть самым жёстким.

**Как исправить:**
Убрать оценочную формулировку и зафиксировать границу наблюдаемым образом. Например:

> Переработка доменной модели PostgreSQL, не необходимая для хранения ссылки на уже сохранённый blob, не входит в рамки этой задачи.

Или ещё жёстче:

> Проектирование отдельной модели хранения blob в PostgreSQL не входит в рамки этой задачи; задача ограничена сценарием, в котором приложение может сослаться в метаданных на объект, уже сохранённый во внешнем object storage.

## Итог

Есть замечания. Блокирующих дефектов по внутренней логике brief не обнаружено, но раздел `Что не в скоупе` стоит сделать строже, чтобы убрать остаточную неоднозначность границы.
