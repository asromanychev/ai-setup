# Project Problem Statement

## Проблема

AI-агентам и аналитическим пайплайнам нужен воспроизводимый, изолированный сбор сырых данных из внешних каналов и источников. Сегодня каждый потребитель решает это ad-hoc: пишет собственный код забора, не имеет стандартного lifecycle задания, не получает буферизованный результат и не может управлять cleanup по TTL или сигналу.

## Целевые пользователи

| Сегмент | Job To Be Done |
| --- | --- |
| AI-агенты / оркестраторы | Создать `CollectionTask`, получить сырой контент, сигнализировать о потреблении |
| Аналитические пайплайны | Получить воспроизводимый snapshot канала без знания деталей транспорта |
| Команды разработки | Подключить новый источник через изолированный плагин, не трогая ядро |

## Top-Level Outcomes

1. **Буферизованный ingestion** — сервис принимает задание, собирает данные, хранит до потребления, удаляет после сигнала или по TTL.
2. **Изоляция источников** — каждый источник реализован как плагин; ядро не знает о деталях транспорта.
3. **Воспроизводимость** — один и тот же `CollectionTask` при повторном запуске даёт детерминированный результат для одного временного окна.

## Out Of Scope (проект-уровень)

- Нормализация, embeddings, RAG, семантический анализ.
- UI и пользовательский интерфейс.
- Приватные источники, авторизованные каналы.
- S3-загрузка вложений (MVP: заглушка).
