# Cross-references (code ↔ docs)

Цель: поддерживать двустороннюю навигацию:

- из кода к canonical документу (спек, ADR, feature),
- из документации к реализации и тестам.

## Code → docs

Модуль, реализующий задокументированную логику, содержит комментарий-ссылку на canonical документ.

Минимальный контракт:
1. Ссылка указывает относительный путь от корня репозитория.
2. Аннотация объясняет, какой аспект документа релевантен данному модулю.

Пример (Ruby):
```ruby
# Контракт плагина: memory-bank/issues/0001-plugin-contract/spec.md
# Инвариант sync_step: section "Plugin Contract"
module Collect
  class Plugin; end
end
```

## Docs → code

В документации допускаются ссылки на файлы и строки (после появления кода). Каждая ссылка аннотирована: что по ссылке и зачем читать.

Пример:
```
Реализация: [`core/lib/collect/plugin.rb`](../core/lib/collect/plugin.rb) — базовый класс, контракт `sync_step`.
```

## Каноническая точка входа для агента

Если агент ищет реализацию — смотреть `index.md` раздел «Быстрая навигация: канон вне `memory-bank/`».  
Если агент ищет документ по коду — смотреть комментарий в заголовке файла или модуля.
